## Design Context

### Users
Avalonia and .NET developers evaluating ShadUI or looking for concrete component implementation examples. They use the demo as a component workbench: browsing controls, comparing states, testing interactions, and learning expected behavior.

### Brand Personality
Precise, practical, polished.

### Aesthetic Direction
Preserve the restrained desktop ShadUI demo aesthetic in both light and dark themes. The browser version should feel like the same product, not a marketing microsite, while adapting navigation and density for desktop, tablet, and touch-sized browser windows.

### Design Principles
- Keep the desktop and browser component catalogs structurally consistent.
- Prioritize discoverability, real interaction states, and useful examples over decoration.
- Adapt navigation and layout without hiding core functionality on smaller screens.
- Use compiled bindings and browser-safe platform services.
- Preserve ShadUI’s existing typography, spacing, color, and control-theme language.
